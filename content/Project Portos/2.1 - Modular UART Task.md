While the initial [[2 - UART Communication|UART program]] was a good learning experience, there was still a lot to be improved. 

The main issue is that the UART task is hard-coded to receive input from a specific UART instance and do specific work on the received data. So if we had another device communicating to the Pico via UART, then we'd need to write a new task. It would be much cleaner if I could create a more generic UART task that accepts input parameters, so for every new UART input device I'd just need to create a new instance of the same task. 

# Task Input Parameters
Every UART task receives 3 input parameters:
1. **UART instance**: Either `uart0` or `uart1`
2. **Interrupt handler**: Pointer to a function that handles UART interrupts (i.e. is triggered when there's data in the UART buffer)
3. **Process function**: Pointer to a function that process the data in the UART buffer

Here's a simple diagram of how the Task works:

![[Pasted image 20251030154635.png]]

Now the task is much more modular and can handle different kinds of UART inputs. 

Here's how the input parameters are assigned in the code:
```c
TaskHandle_t my_uartTaskHandle = NULL;

uintptr_t uart_params[] = {(uintptr_t) uart0, (uintptr_t) my_uart_irq_handler, (uintptr_t) my_uart_process};

xTaskCreate(vUARTTask, "My UART Task", 512, (void*) my_uart_params, 1, &my_uartTaskHandle);
```

And the task itself:
```c
void vUARTTask(void* pvParameters) {
    irq_handler_t uart_irq_handler; 
    Proc uart_process;
    uint32_t input;
	
	//Read parameters
    uart_inst_t* uart_inst= (uart_inst_t *) ((uintptr_t*)pvParameters)[0];
    uart_irq_handler = (irq_handler_t) ((uintptr_t*)pvParameters)[1];
    uart_process = (Proc) ((uintptr_t*)pvParameters)[2];
    
    //Set interrupt handler
    if (uart_inst == uart0){
        irq_set_enabled(UART0_IRQ, true);
        uart_set_irqs_enabled(uart0, 1, 0);
        irq_set_exclusive_handler(UART0_IRQ, uart_irq_handler);
    } else if (uart_inst == uart1){
        irq_set_enabled(UART1_IRQ, true);
        uart_set_irqs_enabled(uart1, 1, 0);
        irq_set_exclusive_handler(UART1_IRQ, uart_irq_handler);
    }

    
    for (;;){
	    //Wait for notification from callback
        xTaskNotifyWait(0, 0, &input, portMAX_DELAY);
        //Call process function
        uart_process();
    } 

}
```
# Using Queues for Storing UART Buffer Data
Another issue is how the Process Function reads the data in the UART buffer. Originally, I had the UART interrupt handler send a notification to the Task **every time a new character was read**, with the character attached to the notification. But this felt a bit clunky, as this meant the Task would have to be woken up many times for particularly long strings of data.

I decided to use a **message queue** to store the UART buffer data instead. 

The interrupt handler would populate the message queue with the characters read from the UART buffer. It would notify the Task only once it detected a symbol signifying the end of the message (like a carriage-return/line-feed). Then the Task would call the process function, which would empty the message queue and process the data. 

This way, the Task only gets notified once for every UART message.

![[Pasted image 20251030154702.png]]

Here's how the queue is declared and created in the main program:
```c
#include "queue.h"
QueueHandle_t my_uart_queue = NULL;
...
void main(){
	//Create a queue that can hold 256 characters
	//Each item in the queue is a single character
	my_uart_queue = xQueueCreate(256, sizeof(char));
}
```

This is how the interrupt handler sends characters to the queue and notifies the Task when the message is complete:
```c
void my_uart_irq_handler(){
	char input_char;

    while (uart_is_readable(my_uart_inst)) {
        input_char = uart_getc(my_uart_inst);
        //Check if character is equal to END_SYMBOL
        if (input_char != END_SYMBOL){
	        //Send character to back of Queue
	        xQueueSendToBackFromISR(
	            my_uart_queue, 
	            (void*)&input_char, 
	            NULL);
        } else
        {
	        //Notify UART Task
            xTaskNotifyFromISR(at_uartTaskHandle, 0, eNoAction, NULL);
        }
    }
}
```

And finally, this is how the process function reads from the queue after being called by the Task:
```c
void my_uart_process(uintptr_t input){
    char* start; //Pointer to start of message
    char* end; //Pointer to end of message. Will increment by 1 for every character read from queue
 
    int length = 0;

    start = (char*)malloc(256 * sizeof(char));
    memset(start, 0, length);
    end = start;

	//Read and empty the entire queue
    while(uxQueueMessagesWaiting(at_queue) != 0){
        xQueueReceive(
            my_uart_queue, 
            end++, //Move pointer forward by 1 for every character read
            0
        );
        length++;
    }

    //UART data is now saved in character pointer "start"
	//Process data
	
    free(start);
```
