# Initialize Modem by sending AT commands from Pico to CC1312R

I worked on a test program that needs to accomplish the following:
1. Get **PHY** `AT+PHY?` and set to 1 `AT+PHY=1` (if not already set to 1)  
2. Get **PAN ID** `AT+PAN?` and set to 0xFAA `AT+PAN=0xFAA` (if not already set to 0xFAA)
3. Get **Mode** `AT+MODE?` and set to 2 `AT+MODE=2` (if not already set to 2), where Mode 2 means that the device is acting as modem
4. If any changes were made, reboot the machine by sending `AT+RST` and repeat from Step 1
5. Once the 3 values (**PHY**, **PANID** and **Mode**) have been verified, tell the Modem to join the Personal Area Network (PAN) by sending `AT+JOIN`
6. Wait for the response from `AT+JOIN`, which is an asynchronous response meaning it will arrive at an unknown time. Once it arrives, check the response to verify that the Modem has successfully joined the PAN and save its node address in a variable
