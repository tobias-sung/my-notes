# High and Low
I finally figured out why I was having so much trouble with setting HIGHs and LOWs on the GPIO pins. It's because I was using `gpio_pull_up()` and `gpio_pull_down()` instead of `gpio_set()`! 

You see, each GPIO pin can only use EITHER the pull-up or pull-down resistor, not both! That's why a GPIO pin that can pull down, can't be pulled up again unless you do a `gpio_deinit()`. And vice versa for a GPIO pin using the pull-up resistor. I was able to hack my way to success when resetting the CC1312R by doing an init and deinit, but really that was wrong! Or, I guess it was right in the sense that it worked, but it wasn't the ideal way to do it.

`gpio_set(gpio, 1)` will set a GPIO pin to be high. `gpio_set(gpio,0)` will *clear* the GPIO pin (which is the same as `gpio_deinit()`?).
