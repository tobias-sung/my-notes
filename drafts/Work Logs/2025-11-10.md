# Notify AT Task
After looking through `node.py`, I found the structure of an AT-SEND command to be as follows:
```
AT+SEND=AABB,0,DATA
```
where AABB is the address of the collector, and DATA is the data I want to send in hex format. 

For example, if the radar's min and max values are both 1, then I would send:
```
AT+SEND=AABB,0,11
```

Then the collector, upon receiving, will output:
```
*DATA: 0x4,0xaabb,-61,1 00
```

I'm only supposed to do this if the min/max values have any change, so I created global variables for both to keep track (also so that the AT Task could easily access them when it has to send to the collector).

```c
//Check if minimum/maximum values have changed

if (msg[DATA_OFFSET] != radar_min || msg[DATA_OFFSET+1] != radar_max){

	//Update minimum and maximum  global values
	
	radar_min = msg[DATA_OFFSET];
	
	radar_max = msg[DATA_OFFSET + 1];
	
	//Notify AT Task to send min/max values to node
	
	if (node_state == JOINED_STATE){
		xTaskNotify(at_uartTaskHandle, 1, eSetValueWithOverwrite);
	}

}
```

Then in the `uart_process()` of AT Task:
```c
//Check if the notification came from Radar UART or from UART Interrupt Handler

if (notification_value == 1){

	//Notification came from Radar UART task
	
	char minmax_str[3];
	
	sprintf(minmax_str, "%X%X", radar_min, radar_max);
	
	char at_send_cmd[32];
	
	memset(at_send_cmd, 0, 32);
	
	strcat(at_send_cmd, "AT+SEND=");
	strcat(at_send_cmd, COORD_ADDR);
	strcat(at_send_cmd,",0,");
	strcat(at_send_cmd, minmax_str);
	strcat(at_send_cmd, "\r\n");
	
	debug_print("Sending AT Command - %s", at_send_cmd);
	
	send_at(at_send_cmd);
}
```
