# WrMulti
After a lot of faffing about, I finaly got around to following up on yesterday's finding. That is, there's an extra STOP condition between the register address write and the actual data write when using PIO (whereas when using the I2C hardware interface, the data write starts right after the register address write with no stop condition). I thought it was too simple to be the correct solution, but I still went ahead and did it anyway just to get some work done today.

Eventually I settled on a pretty inelegant solution of removing the STOP condition (although it would have been even messier if I had to import all the PIO I2C functions into "platform.c"). I added two new functions, `pio_i2c_write_blocking_nostop()` and `pio_i2c_write_blocing_nostart()`. They do exactly what their names say. One does performs an I2C write without sending a STOP condition at the end (this is for sending the register address) and the other performs an I2C write without a START condition at the beginning (this is for senidng the firmware data). 

To write both functions, I just copied the code from `pio_i2c_write_blocking()` and removed a few lines of code (for `nostart()`, remove the code that sends the START condition, and same for `nostop()`). 

```c
int pio_i2c_write_blocking_nostop(PIO pio, uint sm, uint8_t addr, uint8_t *txbuf, uint len){
    int err = 0;
    pio_i2c_start(pio, sm);
    pio_i2c_rx_enable(pio, sm, false);
    pio_i2c_put16(pio, sm, (addr << 2) | 1u);
    while (len && !pio_i2c_check_error(pio, sm)) {
        if (!pio_sm_is_tx_fifo_full(pio, sm)) {
            --len;
            pio_i2c_put_or_err(pio, sm, (*txbuf++ << PIO_I2C_DATA_LSB) | ((len == 0) << PIO_I2C_FINAL_LSB) | 1u);
        }
    }
    //Skip Stop
    if (pio_i2c_check_error(pio, sm)){
        err = -1;
        pio_i2c_resume_after_error(pio, sm);
        pio_i2c_stop(pio, sm);
    }
    return err;
}

int pio_i2c_write_blocking_nostart(PIO pio, uint sm, uint8_t addr, uint8_t *txbuf, uint len){
    int err = 0;
    //Skip start
    while (len && !pio_i2c_check_error(pio, sm)) {
        if (!pio_sm_is_tx_fifo_full(pio, sm)) {
            --len;
            pio_i2c_put_or_err(pio, sm, (*txbuf++ << PIO_I2C_DATA_LSB) | ((len == 0) << PIO_I2C_FINAL_LSB) | 1u);
        }
    }
    pio_i2c_stop(pio, sm);
    pio_i2c_wait_idle(pio, sm);
    if (pio_i2c_check_error(pio, sm)) {
        err = -1;
        pio_i2c_resume_after_error(pio, sm);
        pio_i2c_stop(pio, sm);
    }
    return err;

}
```

Then, in `WrMulti()` (which has now has much fewer lines of code since all PIO functions are tucked away in "pio_i2c.c"):
```c
uint8_t WrMulti(
		VL53L5CX_Platform *p_platform,
		uint16_t RegisterAdress,
		uint8_t *p_values,
		uint32_t size)
{
	uint8_t tmp[2] = {(uint8_t)(RegisterAdress >> 8), (uint8_t)(RegisterAdress & 0xFF)};

	pio_i2c_write_blocking_nostop(pio, sm, p_platform->address, tmp, 2);

	pio_i2c_write_blocking_nostart(pio,sm, p_platform->address, p_values, size);


	return 0;
}
```
